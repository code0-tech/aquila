## What is Aquila?

Aquila runs inside the execution block and acts as the gateway to external services. Any request that enters or leaves the execution environment is routed through Aquila.

Aquila keeps internal workflows up to date so the execution service knows what is happening. It also communicates with actions to receive their configurations, schemas, and related metadata.
