## What is Aquila?

Aquila runs inside the execution block and serves as the gateway for external runtime communication.
Requests entering or leaving the execution environment are routed through Aquila.

Aquila keeps flows and runtime definitions synchronized so execution services stay current.
It also communicates with Actions to receive configuration, schema, and related metadata updates.
